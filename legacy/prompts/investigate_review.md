# INVESTIGATE Review Prompt

Issue ${issue_url} 의 `## Bugfix agent INVESTIGATE` 섹션를리뷰해 주세요.

## 태스크

1. `gh issue view` 로 최신의 Issue 本文를 취득
2. INVESTIGATE 스테이트의 필수아웃풋를확인
3. DETAIL_DESIGN 스테이트に移행가능か판정

## 완료조건체크리스트

| # | 항목 | 확인내용 |
|---|------|----------|
| 1 | **再現절차와証跡** | 실행한스텝와결과이기재され, 証跡파일로의참조이 있다 |
| 2 | **期待값와의差** | 期待한다挙動との乖離이 명확에기재되어 있다 |
| 3 | **원인임시説** | 가능性의 있다원인이근거(로그や該当코드)と共に列挙되어 있다 |
| 4 | **보충정보** | 설계에役立つ추가정보이 있다(없이ても가능) |

## 금지事項

**次스테이트이후의책무를신규에실행하지 않는다**
- 예：설계생성, 구현案の提示

**既에 완료한책무를재실행하지 않는다**
- 예：再現を最初부터やり直す, 조사를재실행

※ 기재내용의검증(정합성체크, 모순指摘)는**허가**されてい합니다.

## 출력형식

```markdown
### INVESTIGATE Review Result

#### 검증내용
- <실시한검증와결과를구체적에기재>

#### 체크리스트
- 再現절차와証跡: <OK/NG + 구체적근거>
- 期待값와의差: <OK/NG + 구체적근거>
- 원인임시説: <OK/NG + 구체적근거>
- 보충정보: <OK/NG or N/A>

## VERDICT
- Result: PASS | RETRY
- Reason: <판정이유>
- Evidence: <구체적한 판단근거(Issue本文の該当箇所, 확인한내용)>
- Suggestion: <RETRY時: 구체적한 추가조사指示>
```

## 판정가이드ラ인

| 상황 | VERDICT | 다음스테이트 |
|------|---------|-------------|
| 필수항목이揃い, 모순·부족이 없다 | PASS | DETAIL_DESIGN |
| 부족·모순가 있り再조사이필요 | RETRY | INVESTIGATE |

## レポート방법

1. `gh issue comment` 로 Issue 에 코멘트投稿(VERDICT판정)
2. **PASS판정시만**: 공통규칙에 따라 `gh issue edit` 로 Issue 本文를 업데이트
