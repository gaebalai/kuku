# 무관한 문제의 보고 규칙(공통)

작업 중(테스트 실행・코드 리뷰・문서 확인 등)에, 자신의 변경과 무관한 문제를 발견한 경우의 보고 절차.

## 대상이 되는 문제

- 테스트 실패・에러
- 코드의 불구합・데드 코드・로직 오류
- 문서의 불정합・오기

## 판정 기준

`git diff main...HEAD` 의 변경 차분에 포함되는 파일・모듈과 관계가 없는 문제를 「무관한」으로 판정한다.

판정에 망설이는 경우는 「무관한」이라고 단정하지 않고, 기존 Issue 검색 후에 보고한다(과소 보고보다 과대 보고를 선택한다).

## 보고 플로우

1. 기존 Issue 를 검색(실패 테스트명・에러 메시지・문제의 요약으로 검색)
2. 기존 Issue 가 있는 경우 → 코멘트로 추가 기재(재현한 취지와 환경 정보)
3. 기존 Issue 가 없는 경우 → 신규 Issue 를 생성

## 보고 시의 필수 항목

### 테스트 실패의 경우

- 실패 테스트명(예: `tests/test_foo.py::test_bar`)
- 에러 로그(traceback)
- 재현 커맨드(예: `pytest tests/test_foo.py::test_bar -v`)
- 발견 시의 브랜치・커밋

### 코드・문서의 문제의 경우

- 문제 개소(파일 경로・행 번호・심볼명)
- 문제의 개요와 근거
- 발견 시의 브랜치・커밋
