# Testing Convention

테스트 규약. 설계·구현·리뷰 시에 참조한다.

## 테스트 사이즈 정의

테스트는 리소스 제약(외부 의존의 유무)에 의해 S/M/L의 3사이즈로 분류한다.

| 사이즈 | 명칭 | 특징 | 실행 속도 |
|--------|------|------|---------|
| **S** | Small | 외부 의존 없음. 순수한 로직·밸리데이션·매핑 | 고속(ms) |
| **M** | Medium | 파일 I/O·DB·내부 서비스와의 결합 | 중속(초) |
| **L** | Large | 실 API·E2E·외부 서비스 소통 | 저속(초~분) |

### 마커

```python
@pytest.mark.small
@pytest.mark.medium
@pytest.mark.large
```

### 판정 기준

```
외부 API / 실 서비스 소통 있음 → Large
DB / 파일 / 내부 서비스 결합 있음 → Medium
그 이외(순수 함수·목 완결) → Small
```

## S/M/L 전 사이즈 구현의 원칙

> **설계 단계에서 전 사이즈를 정의하고, 생략의 여지를 배제한다.**

설계서의 "테스트 전략"에는 Small / Medium / Large 각각의 검증 대상을 반드시 기재할 것.
구현 시에도 마찬가지로 전 사이즈의 테스트를 생성한다.

## 스킵 판정 기준

스킵이 허용되는 것은 **물리적으로 생성 불가**인 경우만.

| 허용되는 스킵 이유 | 예 |
|----------------------|-----|
| 스키마 미정비로 Medium을 쓸 수 없다 | DB 테이블이 존재하지 않는다 |
| API 접근 수단이 존재하지 않는다 | 외부 서비스의 샌드박스가 제공되지 않는다 |

### 부당한 생략 이유 (자동적으로 Changes Requested)

이하는 "물리적 불가능"에 해당하지 않기 때문에, 스킵은 인정되지 않는다:

| 부당한 이유 | 대응 |
|------------|------|
| "실행 시간이 길다" | 샘플 기간 등으로 단축 가능 |
| "API 키가 없다" "DB가 기동하지 않고 있다" | 환경 불비는 수정 대상. 사용자에게 상담하거나 스스로 해결 |
| "Small 테스트로 충분히 커버되어 있다" | 테스트 사이즈마다 검증 대상이 다르다 |
| "경미한 변경이므로 테스트 불필요" | 변경의 대소는 스킵 이유가 되지 않는다 |
| "Large 테스트는 스테이징 환경으로 검증" | CI로 실행할 수 있는 구성으로 할 것 |

## AI의 테스트 생략 경향에 대한 경고

> **AI에는 M/L 테스트를 생략하는 강한 경향이 있다. 이 경향을 자각할 것.**
>
> "이 테스트는 불필요하지 않은가?" "Small로 충분하지 않은가?"라고 생각한 경우, **그것은 거의 확실하게 오류이다.**

### 금지 사항

- ❌ Large 테스트를 "스테이징 환경에서의 검증"으로서 뒤로 미루기
- ❌ "실행 시간이 길다"를 이유로 M/L 테스트를 생략
- ❌ "API 키가 없다" 등의 환경 불비를 이유로 M/L 테스트를 스킵
- ❌ "Small 테스트로 충분히 커버되어 있다"고 판단하여 M/L을 생략
- ❌ "경미한 변경"을 이유로 테스트 사이즈를 낮춤

## 설계서 테스트 전략 섹션의 작성법

```markdown
## 테스트 전략

> **CRITICAL**: S/M/L 모든 사이즈의 테스트 방침을 정의할 것.

### Small 테스트
- (검증 대상을 열거: 단체 로직, 밸리데이션, 매핑 등)

### Medium 테스트
- (검증 대상을 열거: DB 연계, 내부 서비스 결합 등)

### Large 테스트
- (검증 대상을 열거: 실 API 소통, E2E 데이터 흐름 등)

### 스킵하는 사이즈 (해당하는 경우만)
- 사이즈: (물리적으로 생성 불가인 이유를 명기. "실행 시간" "환경 의존"은 부당)
```

## 리뷰 시 체크리스트

```
- [ ] Small 테스트의 검증 대상이 정의되어 있는가
- [ ] Medium 테스트의 검증 대상이 정의되어 있는가
- [ ] Large 테스트의 검증 대상이 정의되어 있는가
- [ ] 스킵하는 사이즈가 있는 경우, 이유는 물리적 불가능에 해당하는가
```
